Neues vom QGIS Server und -Webclient

Letztes Jahr haben wir an der FOSSGIS Konferenz gezeigt, wie man problemlos Webkarten mit dem QGIS Server und -Webclient publiziert. Seither sind noch mehr Funktionen dazugekommen, die über den WMS Standard hinausgehen. Häufige Funktionen bei Webkarten sind Objektsuche nach Attributen sowie das Highlighten der Resultate. Bisher kommt man nicht umhin, dafür neben dem eigentlichen WMS noch weitere Serverkomponenten zu installieren und zu konfigurieren. Während dem letzten Jahr wurden daher in QGIS Server Erweiterungen entwickelt, die es erlauben, Suche und Selektion mit dem WMS Server zu machen. So kann die GetFeatureInfo Anfrage ohne Klickkoordinate und mit FILTER-Parameter gemacht werden, worauf der QGIS Server Attributinformationen der passenden Objekte sowie deren Boundingboxen zurückschickt. Die so erhaltenen Objekte können dann in einer GetMap Anfrage mit dem neuen Parameter SELECTION markiert werden. Der QGIS webclient unterstützt diese Art der Suche bereits und kann mit minimalem Aufwand konfiguriert werden.

Ein weiterer Schwerpunkt dieses Vortrags ist das Publizieren von Geodaten und Karten über die QGIS cloud.
